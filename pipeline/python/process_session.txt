PROCESS SESSION DATA.

----------------------------------------------------------------------
1.  Make sure all raw data is organized by FOV, acquisition, run:
----------------------------------------------------------------------
<rootdir>/<animalid>/

----------------------------------------------------------------------
2.  Set processing params for each run:
----------------------------------------------------------------------

python set_pid_params.py -iJR063 -S20171128_JR063 -AFOV2_zoom1x -rscenes -sraw -traw --bidi --motion -f4

python set_pid_params.py -iJR063 -S20171128_JR063 -AFOV2_zoom1x -rgratings_static -sraw -traw --bidi --motion -f2

python set_pid_params.py -iJR063 -S20171128_JR063 -AFOV2_zoom1x_volume -rgratings_static -sraw -traw --bidi --motion -f5

----------------------------------------------------------------------
NOTE:  Once params are created for a given run, that run can be processed on its own with:

python preprocessing/process_raw.py -iJR063 -S20171128_JR063 -AFOV2_zoom1x -rscenes -p123456, 
    where 123456 is the 6-char hash assigned to the processing parameter set.
----------------------------------------------------------------------


----------------------------------------------------------------------
3.  Process each run in the session with associated parameter sets:
----------------------------------------------------------------------

python create_pids_for_session.py -iJR063 -S20171128_JR063 --indie

python process_pids_for_session.py -iJR063 -S20171128_JR063 --zproject

----------------------------------------------------------------------
NOTE:  All process request saved to <session_dir>/tmp_spids/ will be run in parallel.
Processes that exit without errors will move param info files to the 'completed' sub-directory if the tmp_pid (and tmp_spid) directories.
